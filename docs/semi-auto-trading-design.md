# 半自動売買 業務フロー システム化 設計案

クライアント要件（2026-07-13受領・チャット履歴）＋要件定義書 Phase3（車両進捗）。
単なる画面追加ではなく、**半自動売買の業務フロー全体の自動化**。大型フェーズ。
実装前にこの設計をクライアント確認 → 段階実装 → 各段階DB検証。

---

## 0. 全体像（1案件のライフサイクル）

```
[前提] 加盟金・仕入れ資金（預かり金）を本部が管理（CRM/財務）
   │
   ▼
① 仕入れ資金デポジット（預かり金 > 0）
   │  ← オーダー金額は「預かり残高より低い」ことが条件（超過オーダー禁止）
   ▼
② 車両オーダー送信（半自動フローのみ・オンボ完了後）
   │  計算書エビデンス（仕入計算書）を格納（D&D/DL）
   ▼ 自動遷移
③ 仕入れ中（あらゆる仕入進捗を一本化。本部の一件一件操作は不要＝自動）
   │  ← 任意：修理/整備が必要 → 手動で「商品化中」へ（加盟者/本部どちらからも）
   ▼
④（任意）商品化中：整備・修理。作業履歴・費用内訳・エビデンスを記録
   │
   ▼
⑤ 陸送（加盟者費用負担）：陸送費を自動計算（都道府県マスタ／特殊車は個別見積）
   │
   ▼
⑥ 受領ボタン押下（加盟者）→ 取引終了
   │  自動精算：預かり金 − 仕入費 − 商品化費 − 陸送費 = 残金
   ▼
⑦ 取引履歴サマリーに記録／進捗リセット／新規オーダー可能状態に復元
       （残金は次回仕入れへ。加盟者は販売して資金を再デポジット→再ループ）
```

フローチャートは **初回=オンボーディング用**、**以降=半自動売買の進捗管理として何度でもリループ**（クライアント確認済み）。

## 1. 進捗ステータス（車両1案件）
要件定義書＋クライアント指定。大きく分けて：

| ステータス | 意味 | 遷移方法 |
| --- | --- | --- |
| `ordered`（車両オーダー） | オーダー送信直後 | 自動（送信で生成） |
| `sourcing`（仕入れ中） | あらゆる仕入進捗を一本化 | オーダー送信で**自動遷移** |
| `prepping`（商品化中）※任意 | 整備・修理 | 加盟者/本部が**手動**で移行（クロスセル） |
| `delivered`（納品完了/取引終了） | 受領ボタン押下 | 加盟者が**受領**で自動遷移 |

- 「商品化中」を経由しない場合：仕入れ中 → 受領 → 取引終了。
- 「商品化中」の操作画面に**作業履歴・金額内訳**を反映。

---

## 2. 預かり金・資金管理（CRM/財務）

### 2-1 個別管理＋全体管理
- **個別**：加盟店ごとに「加盟金 支払状況」「仕入れ資金（預かり金）支払状況・残高」を管理。
- **全体**：全加盟店の預かり金・支払状況を一覧集計。
- 実装：`portal.member_ledger`（加盟店の預かり金台帳）＋ 支払イベント（deposit/withdraw/settlement）。

### 2-2 超過オーダー制限（自動）
- **車両オーダー金額 < 預かり残高** を条件に送信可能。超過は禁止。
- createOwnOrder に「残高チェック」ガードを追加（既存の semi/オンボ/古物商ガードに追加）。

---

## 3. 自動精算（受領時）
```
残金 = 預かり金
      − 仕入れにかかった費用（計算書エビデンスの金額）
      − 商品化にかかった費用（商品化中で記録した内訳）
      − 陸送費（都道府県マスタ／特殊車は個別見積）
```
- 受領ボタン押下で自動計算 → 取引履歴に記録 → 残金を預かり金残高に反映（相殺）。
- 各費用にエビデンス（計算書・整備明細）を D&D/DL で格納。

---

## 4. 陸送費マスタ（本部・新タブ）
- 本部管理画面に「陸送費設定」タブ新設。**47都道府県のデフォルト陸送費**を設定。
- 陸送先住所（都道府県）と紐付けて自動計算。
- **特殊車対策**：車種メーカーが高級/規格外カテゴリ（フェラーリ/ベントレー/ポルシェ等）の場合、
  自動計算せず「**個別見積もり**」に切り替え（フラグ制御）。メーカーのカテゴリ判定テーブルを持つ。

## 6. データモデル案（migration 022〜）
| テーブル | 用途 |
| --- | --- |
| `member_ledger` | 加盟店の預かり金台帳（残高） |
| `ledger_entries` | 入出金・精算の明細（deposit/settlement/adjust、金額、種別、エビデンス） |
| `vehicle_deals` | 車両1案件（member_id, order_id, status, 仕入金額, 商品化費, 陸送費, 残金, 各日付） |
| `deal_costs` | 案件ごとの費用内訳（仕入/商品化/陸送、金額、note、エビデンスpath） |
| `deal_evidences` | 案件のエビデンス（計算書・整備明細・D&D格納） |
| `shipping_rates` | 47都道府県のデフォルト陸送費 |
| `special_vehicle_makers` | 個別見積対象のメーカー/カテゴリ |
| 既存 `orders` 拡張 | order→deal への発展、金額と残高連動 |

`payments`（既存）に加盟金/月額があるので、加盟金支払状況はそれを活用しつつ ledger と整理。

## 7. フェーズ分割案（各フェーズ = migration + 画面 + 検証）
1. **預かり金台帳＋CRM資金管理**：member_ledger/ledger_entries、CRMに加盟金・仕入資金の個別/全体管理。
2. **超過オーダー制限**：オーダー金額 < 預かり残高 のガード。
3. **車両案件ライフサイクル**：vehicle_deals＋進捗ステータス自動遷移（オーダー→仕入れ中）＋受領ボタン→取引終了。
4. **商品化中・費用内訳・エビデンス**：deal_costs/deal_evidences、作業履歴・金額内訳・D&D。
5. **陸送費マスタ＋自動計算＋特殊車個別見積**：shipping_rates/special_vehicle_makers、住所紐付け。
6. **自動精算＋取引履歴サマリー＋リセット**：受領で残金計算・履歴記録・進捗リセット・新規オーダー復元。
7. **ダッシュボード進捗ボード（横軸）**：最上部に進捗可視化。AI/収益グラフは次フェーズ。

## 8. 確定事項（クライアント回答 2026-07-14）

- **Q1 進捗ステータス**：4段確定 `ordered`(車両オーダー)→`sourcing`(仕入れ中)→`prepping`(商品化中・任意)→`delivered`(納品完了)。「仕入れ中」は全仕入進捗を一本化。
- **Q2 仕入費の金額**：手入力に加え、**「手離れのいい方法」をもう1つ追加**。→ 計算書エビデンスから金額を取り込む方式（当面は「エビデンス添付＋金額を確認して確定」の半自動、将来OCR/AI抽出）。2手段を用意（手入力／エビデンス取込）。
- **Q3 陸送費マスタ**：一律ではなく**発地×着地を個別に設定できるマスタ**。都道府県ペアで料金設定（未設定ペアはデフォルト or 個別見積）。
- **Q4 特殊車**：初期リスト（フェラーリ/ベントレー/ポルシェ/ランボルギーニ/ロールスロイス/マクラーレン/アストンマーティン）で開始＋**随時追加**（本部画面）。
- **Q5 商品化スキップ**：`sourcing` → **受領（受け取り完了スイッチ）** → 取引終了表示 → 履歴自動反映 → **チャートがスタート位置へ遷移**。
- **Q6 受領後**：①取引履歴に記録 ②進捗リセット ③残金を次回繰越 ④新規オーダー可能に復元、で確定。
- **Q7 精算式**：`預かり金 − 仕入費 − 商品化費 − 陸送費 = 残金`。**追加要件**：陸送費の右に**追加項目（任意の費目）を足せる**＋**すべての費目名称を変更できる**（動的な費目リスト＝deal_costs を label 自由・並び追加可能に）。
- **Q8 陸送費外部連携**：将来対応。当面は本部が都道府県ペア料金を手設定。

### 確定に伴う設計変更
- **精算は固定4費目でなく「動的費目リスト」**：deal_costs に `label`（費目名・変更可）・`sort_order`・`kind`（仕入/商品化/陸送/その他）・金額・エビデンス。残金=預かり金−Σ(費目)。
- **仕入費入力は2モード**：手入力 or エビデンス取込（添付→金額確認→確定）。
- **陸送費マスタは (発地県, 着地県) ペア**：shipping_rates(from_pref, to_pref, amount)。特殊車メーカーは shipping を個別見積フラグに。
- **受領スイッチ**：deal を delivered にし、精算実行→ledgerに settlement 記帳→履歴サマリー→進捗リセット→新規オーダー可能。
